75 <h1><a class="anchor" id="autotoc_md1"></a>
76 Requirements:</h1>
77 <p>0. Please Ensure you have the RPI patched with a PREEMPT_RT patch, otherwise you wouldnt be able to apply a RT priority to SCHED_FIFO</p><ol type="1">
78 <li>For using this library, your user will need to have root permissions.</li>
79 <li>Secondly, to set RT priorities, please set ulimit configs in the following way:<ol type="a">
80 <li>Add your user to a group, in this example, called "rtprio"</li>
81 <li>Give this group root permissions:<br />
82  <code>sudo groupadd rtprio</code> <br />
83  <code>sudo usermod -a -G rtprio &lt;username&gt;</code><br />
84  <code>sudo chmod 4750 /usr/bin/chrt</code> <br />
85 </li>
86 <li>Now you have to configure hard and soft priority limits which the group can use:</li>
87 <li>Open /etc/security/limits.conf in an editor of your choice and add the following lines:<br />
88  <code>&lt;username&gt; hard rtprio 99</code> <br />
89  <code>&lt;username&gt; soft rtprio 99</code></li>
90 <li>Reference for the above: <a href="https://stackoverflow.com/questions/10704983/operation-not-permitted-while-setting-new-priority-for-thread">https://stackoverflow.com/questions/10704983/operation-not-permitted-while-setting-new-priority-for-thread</a></li>
91 <li>One way to programatically change soft and hard limit : <a href="https://man7.org/linux/man-pages/man2/getrlimit.2.html">https://man7.org/linux/man-pages/man2/getrlimit.2.html</a> </li>
92 </ol>
93 </li>
94 </ol>
